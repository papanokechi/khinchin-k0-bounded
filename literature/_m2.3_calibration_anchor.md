## 0. TL;DR

The M1.2-frozen hybrid basis is

```
B_D(C)  =  {1, K_0, K_0², ..., K_0^D, log K_0}  ∪  {K_0·c : c ∈ C}
```

with operator-fixed `D ≤ 6` and a height bound `H` to be specified in M2.3, all at 500 dps.

**BBC 1997 §4 has already PSLQ-null-tested the pure-power sub-basis `{1, K_0, K_0², …, K_0^50}` at much stronger parameters (D = 50, H = 10^70, 7350 dps).** Therefore the pure-power portion of B_D(C) at D ≤ 6 / 500 dps is a **strict weakening** of BBC's published null.

Consequently, the M2.3 success predicate **must** target the rows where the present mission produces genuinely new information — namely:

1. The `log K_0` row in isolation against the pure-power coordinates (BBC tested log K_0 inside a 21-dim log-multiplicative basis, NOT against pure powers of K_0);
2. The `K_0·c` bilinear product rows for c ∈ C ⊆ {π, e, ln 2, ζ(2), ζ(3), G, γ, …} (BBC tested log-form Σ a_k log c_k, NOT bilinear K_0·c).

Failure to calibrate against BBC 1997 risks silently re-running a weaker version of BBC's Test 1 and producing a Zenodo deposit at M6 whose central null result is structurally dominated by published 1997 work. **This document is the calibration anchor that prevents that.**

## 3. Proposed M2.3 success-predicate calibration (draft for operator review at M2.3)

> **M2.3 success predicate (calibrated):** The M3.1 PSLQ sweep is _informative_ — i.e., produces information not already published in BBC 1997 — if and only if it reports:
>
> **(A)** a verified integer relation OR a verified null at height-bound H_target ≥ 10^? on **at least one** of:
>   - the `log K_0` row in isolation against {1, K_0, K_0², …, K_0^D}, **OR**
>   - any of the `K_0·c` bilinear rows for c ∈ C
>
> with c chosen from a fixed named-constant basis to be specified in M2.3 §C-set definition.
>
> **(B)** The pure-power sub-basis {1, K_0, K_0², …, K_0^D ≤ 6} is reported as a **BBC-1997 reproduction diagnostic**, not as a primary result.
>
> **(C)** Height-bound H_target should be calibrated against the operator's precision budget (500 dps). A pure-precision argument gives an upper bound: for a basis of dimension k, PSLQ can certify nulls up to roughly H ≈ 10^(precision_digits / k). At 500 dps and basis dim ~10 (B_D(C) with D=6, |C|=3), this gives H ≈ 10^50. This is WEAKER than BBC's 10^70 on pure-power Test 1 (k=51) but stronger than BBC's 10^20 on log-multiplicative Test 2 (k=21). The fact that on the **novel rows** the present mission achieves stronger H than BBC's published nulls (because the basis dimension is smaller) is itself the warrant for a publishable result.

The above is a CLI **proposal**, not a binding M2.3 specification. M2.3 is the milestone where the operator fixes the formal predicate.

### §7.4 Positive predicate (success criterion: relation discovery in the complement)

A run of the M3.1 harness produces a **POSITIVE result** if and only if all of the following hold:

1. **Relation found.** mpmath.pslq returns a non-None coefficient vector `(m_0, m_1, …, m_{14})` at precision P = 2160 dps, dimension n = 15, with `max_i |m_i| ≤ 10⁷⁰`.

2. **Tolerance check.** The residual `|Σ m_i · b_i|` (where `b_i` are the basis elements at precision P) is below the PSLQ default tolerance `2^{-(0.75·prec)}` ≈ 10^{-1620} at P = 2160 dps.

3. **Complement scope.** At least one of the coefficients m_i corresponding to `log K_0` or `K_0 · c_j` (for c_j ∈ C) is non-zero. (I.e., the relation is NOT confined to EF1.)

4. **Cascade stability.** The same relation is recovered at precision 2P = 4320 dps and 4P = 8640 dps, with the same coefficient vector (modulo trivial scalar multiples). PSLQ's known false-positive mode at low precision is suppressed by the cascade.

5. **Excluded-Family checks.**
   - If the relation lies entirely within EF1 (all coefficients on {1, K_0, …, K_0^D} are non-zero AND all coefficients on {log K_0, K_0·c_j} are zero), classify as **BBC-1997-reproduction-diagnostic**, NOT primary positive.
   - If the relation requires EF3 (GKW-method-ceded) machinery for interpretation, **HALT and surface** to operator per `seeds/README.md` DO-NOT-REENTER.
   - Otherwise, positive primary result.

6. **Verification class (per H9).** The M3.1 harness's positive output inherits classification:
   - The PSLQ-detection mechanism is `proven_corollary` (FBA 1999 Theorem 3 gives the overshoot bound on found relations) **with the D2 boundary-case caveat** (mpmath uses γ = √(4/3) exactly; rigorous Theorem-3 needs Bailey 1998 H8 paper-read if M6 cites it).
   - The relation itself is `rigorous_theorem`-class once verified: it satisfies an arithmetic identity exactly modulo a real-arithmetic residual below tolerance, which is theorem-grade for the rational-relation case.

### §7.5 Negative predicate (success criterion: null PSLQ across full basis)

A run of the M3.1 harness produces a **NEGATIVE result** if all of the following hold:

1. **No relation found.** mpmath.pslq returns None across the full B_D(C) basis at precision P = 2160 dps with `maxcoeff = 10⁷⁰` and `maxsteps` raised to ~100,000 (for rigorous-tier convergence; see precision_budget.md §5.2).

2. **Cascade stability.** mpmath.pslq returns None at 2P = 4320 dps and 4P = 8640 dps as well, with the SAME basis. Cascade null is required; transient null at single precision is not sufficient.

3. **Height confidence at TWO TIERS** (per U-MISSION-L two-tier predicate):

   **Tier 1 (empirical, `field_standard_practice` per H9):**
   - H_empirical = BBC-1997-calibrated confidence relation `H ≈ c · 10^{P/(c·n)}` with c ≈ 2.06 at (P=2160 dps, n=15) ⟹ H_empirical = **10⁷⁰**.
   - Interpretation: "No integer relation exists with height ≤ 10⁷⁰ under BBC-1997-style empirical confidence calibration."
   - Verification class: `field_standard_practice` (per H9; BBC/BBP/Bailey-class community-validated empirical practice; NOT theorem-grade).

   **Tier 2 (rigorous, `proven_corollary` per H9):**
   - H_rigorous = max(100 × `reported_norm`, exp((K - 2·n³) / (2·n²))) per FBA 1999 Theorem 1 + Corollary 2.
   - The first term uses mpmath.pslq verbose-mode output (`harness/rigorous_bound.py` parses); the second uses iteration count K.
   - Reported in harness output alongside H_empirical. **If H_rigorous ≥ 10⁷⁰** (requires K ≈ 50,000-100,000 iterations at n=15; see precision_budget.md §5.2), the negative claim is theorem-grade at the BBC-parity height. **Otherwise**, the negative claim is empirical-tier-only at the BBC parity, and rigorous-tier at whatever height H_rigorous actually reached.
   - Verification class: `proven_corollary` (per H9; FBA 1999 Theorem 1 is `rigorous_theorem`; the derived bound 100·norm is its proven-corollary application; modulo divergences D1-D4 documented in precision_budget.md §5.1).

4. **Excluded-Family check on the negative.**
   - Null applies to the full B_D(C) basis.
   - **Reproducing the BBC 1997 null on EF1** (the pure-power subset) at the mission's strictly-weaker (D, P) is treated as the **reproduction diagnostic**; not the primary negative result.
   - **Primary negative result is on the complement** (log K_0 + bilinear K_0·c_j rows): null on these coordinates is the publishable observation.

### §7.6 Predicate failure modes (preserve audit trail)

A run produces NEITHER positive NOR negative — i.e., it FAILS the predicate — if:

- mpmath.pslq returns a relation but cascade fails at 2P or 4P (transient PSLQ false-positive); log as `cascade_failure_event`, do not count as positive.
- mpmath.pslq returns None at P but the wall-clock budget is exhausted before maxsteps complete; log as `precision_or_budget_exhausted`, do not count as negative.
- The relation requires EF3 (GKW) interpretation: HALT and surface (per §7.4 item 5).
- The relation lies entirely within EF1: classify as BBC-reproduction-diagnostic (per §7.4 item 5); not primary.

Failure mode events accumulate in the M3 audit log; they are M6-Discussion content (transparent reporting), not silent suppression.

## 2. What changed

### 2.1 `harness/verify.py` (R1 mechanical fix)

Function `empirical_height(P, n, c=2.06)` (lines 65-86, was lines 65-67):

**Before (defective):**
```python
def empirical_height(P: int, n: int, c: float = 2.06) -> float:
    return 10 ** (P / (c * n))  # OverflowError when P/(c*n) >= ~307
```

**After (R1):**
```python
def empirical_height(P: int, n: int, c: float = 2.06):
    exponent = P / (c * n)
    if exponent < 307.0:
        return 10.0 ** exponent  # native float; preserves M3.2a parity at n=15
    saved_dps = mp.dps
    try:
        mp.dps = max(50, int(exponent) + 20)
        return int(mp.power(mp.mpf(10), mp.mpf(exponent)))  # Python int, no overflow
    finally:
        mp.dps = saved_dps
```

Smoke-tested at three regimes:
- `empirical_height(2160, 15)` = `7.99673347137433e+69` (float, M3.2a parity)
- `empirical_height(2160, 2)` = `int(10^524)` (Python int, no overflow)
- `empirical_height(8640, 2)` = `int(10^2098)` (Python int, no overflow)

Committed at `16aaad6` ("U-MISSION-N R1: mpmath fallback for empirical_height overflow").

def empirical_height_dual(P: int, n: int, maxcoeff_exp: int, c: float = 2.06) -> dict:
